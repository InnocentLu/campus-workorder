# 校园工单管理系统 — 中期开发文档

## 一、题目描述

### 1.1 项目背景

传统校园后勤报修依赖电话、纸质单或微信群沟通，存在以下痛点：报修渠道分散，维修进度不透明，派单依赖人工调度，历史工单难以追溯，学生无法评价维修质量。为解决上述问题，设计并实现一套基于 Web 的**校园工单全生命周期管理系统**。

### 1.2 项目目标

构建一个面向校园后勤场景的工单管理平台，覆盖 **报修提交 → 管理员派单 → 维修工处理 → 用户确认 → 评价归档** 的完整闭环，支持四种角色（学生、教师、维修工、管理员）协同工作。

### 1.3 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + TypeScript + Tailwind CSS | 组件化 SPA，蓝色校园风格，深色模式 |
| 状态管理 | Zustand | 轻量级全局状态（Auth） |
| UI 组件 | shadcn/ui + Lucide Icons | 可定制的基础组件库 |
| 动画 | Framer Motion | 页面过渡、列表入场、交互反馈 |
| 图表 | Recharts | 仪表盘趋势图、饼图 |
| 后端 | Node.js + Express + TypeScript | RESTful API |
| ORM | Prisma | 类型安全的数据库访问层 |
| 数据库 | PostgreSQL (Supabase) | 生产环境云托管 |
| 认证 | JWT (jsonwebtoken + bcryptjs) | 基于 Token 的无状态认证 |
| 部署 | Vercel (前端) + Render (后端) + Supabase (数据库) | 全云托管 |

### 1.4 线上地址

| 后端 API | `https://campus-workorder.onrender.com` |

## 二、问题分析

### 2.1 业务流程分析

校园报修业务涉及四个角色，每个角色的核心需求如下：

```
学生/教师（报修人）
  ├── 提交报修工单（选择工种、描述问题、上传照片）
  ├── 查看工单处理进度（时间轴）
  ├── 确认维修完成或退回重做
  └── 评价维修质量（星级评分）

管理员（调度者）
  ├── 查看待派单工单列表
  ├── 按工种匹配维修工并指派
  ├── 管理用户（创建、编辑、批量导入Excel）
  ├── 管理工种（水电/网络/空调/门窗/墙面/家具）
  ├── 数据仪表盘（统计卡片 + 趋势图 + 饼图）
  └── 发送全站广播通知

维修工（执行者）
  ├── 查看被指派的工单（我的任务）
  ├── 开始处理 → 申请完成
  ├── 填写维修记录（内容、费用、配件）
  └── 设置个人工种和联系方式

系统
  ├── 登录注册（密码强度校验 + 失败锁定）
  ├── 站内通知（铃铛提醒 + 管理员广播）
  ├── 操作日志（全流程可审计）
  └── 深色模式 + 移动端自适应
```

### 2.2 核心问题

1. **派单效率**：管理员需要快速找到与工单工种匹配的维修工，系统需自动按工种筛选并优先展示匹配人员。
2. **进度可视化**：报修人需要清晰了解工单当前处于哪个环节（提交→指派→处理→确认→评价），采用垂直时间轴展示。
3. **确认机制**：维修完成不是终点，必须由报修人确认才算真正完成；若不符合要求可退回重做。
4. **通知触达**：关键节点（新工单、指派、确认请求、退回）需自动推送站内通知，避免信息遗漏。
5. **安全防护**：密码强度要求（8位+字母+数字）、登录失败锁定（5次/15分钟）、角色权限隔离（RBAC）。

### 2.3 工单状态机

```
PENDING ──(管理员指派)──▶ ASSIGNED ──(维修工开始处理)──▶ PROCESSING
                                                              │
                                          (维修工申请完成)      │
                                                              ▼
                                                     PENDING_CONFIRM
                                                       ╱        ╲
                                            (报修人确认)          (报修人退回)
                                                     ╱            ╲
                                               COMPLETED      PROCESSING（重新处理）
                                                    │
                                               (评价)
                                                    │
                                               CLOSED
```
